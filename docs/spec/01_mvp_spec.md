# 遠隔眼底画像診断支援サービスプラットフォーム MVP仕様書

## 1. 目的
本プラットフォームは、依頼元（健診センター等）が登録した受診者情報と眼底画像を、受託側で受付・読影割当・医師読影・QC確認・返却・請求/支払まで一気通貫で処理する。

## 2. 対象範囲（Phase 1: MVP）
- 対象画像: JPEG（将来 PNG/TIFF/DICOM拡張）
- 案件単位: 受診者単位
- 依頼方式: 画面入力 + CSV一括
- 読影方式: Webビューア + 構造化所見
- 返却方式: PDF + ポータル表示
- 会計: 月次締め + 件数帯単価
- 支払: 医師別・依頼元別の単価対応

## 3. ロール
- 依頼元管理者/依頼元一般
- 受付オペレーター
- 読影調整担当
- QC担当
- 医師
- 経理担当
- システム管理者
- 監査閲覧者

## 4. E2E業務フロー（MVP）
1. 依頼元が案件登録（単票/CSV、画像アップロード）
2. 受付オペレーターが整合確認（左右眼、必須項目、不備判定）
3. 調整担当が医師へ割当
4. 医師が読影・所見入力・提出
5. QC担当が体裁/入力漏れ確認（差戻し可）
6. 納品確定し依頼元へ返却（PDF生成）
7. 月次で請求計上・医師支払計上

## 5. ステータス定義
- `DRAFT`
- `UPLOADED`
- `RECEPTION_PENDING`
- `RECEPTION_ACCEPTED`
- `NEEDS_FIX`
- `ASSIGNED`
- `READING_IN_PROGRESS`
- `READING_SUBMITTED`
- `QC_IN_PROGRESS`
- `QC_REWORK`
- `COMPLETED`
- `DELIVERED`
- `INVOICED`
- `PHYSICIAN_PAYABLE`
- `CLOSED`

## 6. 機能一覧
### 6.1 依頼受付
- 案件作成
- 受診者情報登録
- 問診情報登録
- 画像メタ登録
- 画像アップロード（署名URL方式）
- CSV取込

### 6.2 読影運用
- 受付チェック
- 手動アサイン
- 医師読影フォーム
- QCチェックリスト
- 差戻しコメント

### 6.3 納品
- 納品PDF生成
- 納品公開/取消
- 依頼元ポータル閲覧

### 6.4 会計
- 依頼元請求生成
- 医師支払生成
- 件数帯単価適用
- 緊急/再読影加算

### 6.5 管理
- ユーザー管理
- 契約管理
- 料金テーブル管理
- 監査ログ閲覧

## 7. 非機能要件
- 監査可能性: 主要操作を全て監査ログ化
- セキュリティ: HTTPS前提、Cookie HttpOnly/SameSite、RBAC
- 可用性: バックグラウンドジョブでPDF/通知を非同期化
- トークン最適化: 構造化データ主体・差分送信・要約保持

## 8. KPI
- 受付→読影開始時間
- 読影TAT
- 差戻し率
- SLA遵守率
- 再読影率
- 1件粗利
